# codex-feishu-bridge Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a Go single-binary local daemon that lets one allowlisted user start and resume Codex tasks from Feishu WebSocket messages.

**Architecture:** The binary is split into small internal packages: config and doctor prepare runtime settings, transport normalizes Feishu events, router owns authorization and task routing, store persists SQLite state, codexrunner executes `codex exec --json`, notifier renders cards, and app wires the long-running service. Package boundaries follow the spec contracts so each package can be tested with fakes before real Feishu or real Codex is used.

**Tech Stack:** Go 1.22+, standard `flag`/`context`/`log/slog`, `github.com/pelletier/go-toml/v2` for TOML, `modernc.org/sqlite` for CGO-free SQLite, `github.com/larksuite/oapi-sdk-go/v3` for Feishu WebSocket and message APIs, Go `testing` with fakes and temporary directories.

}

	Alias     string
	Command   string
	CWD       string
	Model     string
	Sandbox   string
	Approval  string
	ExtraArgs []string
	LogRetentionDays int
}

func DefaultPath(home string) string
func Load(path string, getenv func(string) string) (Config, error)
func (cfg Config) Validate(getenv func(string) string, stat func(string) error) []Diagnostic

Resolution rules:

- Empty alias uses `Workspace.Default`.
- Project alias overrides `cwd`, `model`, `sandbox`, and `approval`.
- `model` falls back from project `model` to `[codex].default_model`.
- `command` always comes from `[codex].command` after defaulting to `codex` and is included so task creation can snapshot `effective_codex_command`.
- `extra_args` comes from `[codex].extra_args` and is included in the resolved runtime config so tasks can snapshot it.
- `log_retention_days` defaults to `14`, is validated as positive, and is available to app startup pruning.
- Missing default workspace is validation error.
- Unknown alias is validation error.
- `DefaultPath(home)` returns `<home>/.codex-feishu-bridge/config.toml`.
- Empty `Paths.StateDB` and `Paths.LogDir` are resolved from the config user's home directory.
- `Validate` returns structured diagnostics with `Level` (`error`, `warn`, `ok`), `Code`, and `Message`; `doctor` renders these diagnostics.

### Task 7: Implement Transactional Store API

**Files:**

- [ ] **Step 1: Add failing transaction tests**

Cover:

- `AdmitNewTask` records `dedup_key`, creates task, snapshots effective command/cwd/sandbox/model/approval/approval flag support/extra args, and creates initial running run in one transaction.
- `AdmitResumeRun` records `dedup_key`, verifies creator-only continuation, verifies task status allows resume, and creates a running resume run in one transaction.
- `AdmitResumeRun` uses an immediate SQLite transaction so the partial unique index rejects racing resumes before Codex starts.
- `AdmitResumeRun` rejects resume when `tasks.codex_session_id` is empty, because Codex cannot resume a task without a known session id.
- duplicate completed `dedup_key` returns replay without creating a second task or run.
- stuck `processing` dedup rows can be marked failed during recovery.
- `AdmitResumeRun` rejects same-task concurrent active run through `runs_one_active_per_task`.
- `RecordRunSession` persists `tasks.codex_session_id` and `runs.codex_session_id` while the run is still active, and is idempotent for repeated same session ids.
- `FinishRun` updates run fields, task status, `codex_session_id`, and dedup state together.
- `RefreshUsers` upserts configured open ids as enabled and disables ids removed from config; `UserEnabled` reports the local audit/cache state.
- route insert and route resolve work for start cards, result cards, and route misses.
- startup recovery marks stale running runs failed with restart reason, updates latest-run task status, and keeps resumability only when `codex_session_id` exists.
